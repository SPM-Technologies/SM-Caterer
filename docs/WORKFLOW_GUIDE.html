1. System Overview

SM-Caterer is a multi-tenant catering management system designed to help catering businesses manage their operations efficiently.

What is Multi-Tenant?

Multi-tenant means multiple catering businesses (tenants) can use the same application, but each tenant's data is completely isolated. One tenant cannot see another tenant's customers, orders, or data.

Key Features

  • Customer Management
  • Order Management with Wizard
  • Menu & Recipe Management
  • Payment Tracking
  • UPI QR Code Payments
  • Multi-language Support (EN, HI, MR)
  • Reports & Analytics
  • Email Notifications

2. User Roles & Permissions

SUPER_ADMIN Super Administrator

Who: The system owner/developer who manages the entire platform.

Can do:

  • Create and manage Tenants (catering businesses)
  • Create Tenant Admin users for each business
  • Access all tenants' data (for support purposes)
  • System-wide configuration

Cannot do: Day-to-day catering operations (that's for tenant users)

TENANT_ADMIN Tenant Administrator

Who: The owner/manager of a catering business.

Can do:

  • Full access to their tenant's data
  • Create Manager and Staff users
  • Manage all Master Data (Units, Materials, Menus, etc.)
  • Approve/Reject orders
  • View all reports
  • Configure email and payment settings

MANAGER Manager

Who: Senior staff who manages day-to-day operations.

Can do:

  • Create and manage customers
  • Create and manage orders
  • Approve orders (workflow)
  • Record payments
  • View reports
  • Manage master data

STAFF Staff

Who: Regular employees handling basic tasks.

Can do:

  • View customers and orders
  • Create new orders
  • Record payments
  • View basic reports

Cannot do: Approve orders, manage master data, or access sensitive settings

4. Initial Setup (Master Data)

Important: Before creating orders, you must set up your Master Data. This is a one-time setup (with occasional updates).

Setup Order (Do This First!)

# What to Setup Menu Path Description
1 Units Master Data → Units Measurement units like KG, LTR, PCS, PLATE, etc.
2 Materials Master Data → Materials Raw materials/ingredients (Rice, Oil, Vegetables, etc.)
3 Event Types Master Data → Event Types Types of events you cater (Wedding, Birthday, Corporate, etc.)
4 Menus Master Data → Menus Food items you offer (Biryani, Paneer Butter Masala, etc.)
5 Recipes (Optional) Master Data → Recipes Link menus to materials with quantities
6 UPI QR Codes (Optional) Master Data → UPI QR Codes Your UPI payment details for QR code generation

Example: Adding a Unit

1 Go to Master Data → Units

2 Click "Add New" button

3 Enter Unit Code: KG

4 Enter Unit Name: Kilogram

5 Click "Save"

Example: Adding a Menu Item

1 Go to Master Data → Menus

2 Click "Add New" button

3 Enter Menu Code: VEG-BIRYANI

4 Enter Name: Vegetable Biryani

5 Select Category: VEG

6 Enter Cost Per Serve: 150

7 Click "Save"

5. Managing Customers

Adding a New Customer

1 Go to Customers from the sidebar

2 Click "Add Customer" button

3 Fill in the details:

  • Name: Customer's full name
  • Phone: Mobile number (for contact)
  • Email: Email address (for receipts)
  • Address: Full address
  • City, State, Pincode: Location details
  • GSTIN: (Optional) For GST billing

4 Click "Save"

Tip: You can also create a new customer directly while creating an order using the Order Wizard.

6. Creating & Managing Orders

Creating a New Order (Order Wizard)

The Order Wizard guides you through 5 simple steps:

Step 1: Select Customer

1 Go to Orders → New Order

2 Either:

  • Search existing customer by name or phone, OR
  • Create new customer by clicking "New Customer" tab

3 Click "Next"

Step 2: Event Details

Fill in the event information:

  • Event Type: Wedding, Birthday, Corporate, etc.
  • Event Date: When is the event?
  • Event Time: Start time
  • Guest Count: Number of people to serve
  • Venue Name: Where is the event?
  • Venue Address: Full address of venue

Step 3: Select Menu Items

Add food items to the order:

  • Browse available menu items
  • Click "Add" on items you want
  • Set quantity and price per item
  • See running total at the bottom

Step 4: Select Utilities (Optional)

Add extra items like:

  • Tables & Chairs
  • Plates & Cutlery
  • Decoration items
  • Service staff

Step 5: Review & Submit

Review the complete order:

  • Customer details
  • Event details
  • Menu items with prices
  • Utilities with prices
  • Apply discount (if any)
  • Apply tax (if applicable)
  • See Grand Total

Click "Create Order" to save.

Status Meaning Who Can Change Next Action
DRAFT Order just created, not yet submitted Creator (Staff/Manager) Submit for approval
PENDING Waiting for manager/admin approval Manager / Tenant Admin Approve or Reject
CONFIRMED Order approved, ready to start Manager / Tenant Admin Start preparation
IN_PROGRESS Food is being prepared Manager / Tenant Admin Mark as Complete
COMPLETED Order delivered successfully - Collect remaining payment
CANCELLED Order was cancelled Manager / Tenant Admin -

Workflow Actions

To Submit an Order:

  1. Open the order
  2. Click "Submit" button
  3. Order moves to PENDING

To Approve an Order:

  1. Open the pending order
  2. Review the details
  3. Click "Approve" button
  4. Order moves to CONFIRMED

8. Recording Payments

Recording a Payment

1 Go to Payments → Record Payment

2 Select the Order (or Customer)

3 Enter Amount received

4 Select Payment Method:

  • Cash
  • UPI
  • Card
  • Bank Transfer
  • Cheque

5 Enter Transaction Reference (for digital payments)

6 Click "Save"

Auto-calculation: The system automatically calculates:
  • Total Paid: Sum of all payments
  • Balance Due: Grand Total - Total Paid

Payment from Order View

You can also record payment directly from an order:

  1. Open the order
  2. Click "Record Payment" button
  3. Enter payment details
  4. Payment is automatically linked to the order

Common Mistakes to Avoid:

  • Don't create orders without setting up Master Data first
  • Always select the correct customer before creating an order
  • Double-check event date and guest count
  • Record payments promptly to keep balance accurate

Best Practices:

  • Set up all master data before going live
  • Create customers with complete contact information
  • Use meaningful menu codes (e.g., VEG-BIRYANI, NV-CHICKEN)
  • Review pending balance report weekly
  • Export reports monthly for records
